Project

STM32H563 UART Configurable Console

Runtime peripheral reconfiguration via an interactive serial console

STM32H563 Embedded C UART / USART3 STM32 HAL CMSIS Interrupt-driven I/O

Overview

This project turns a basic UART logging setup into an interactive embedded console on the NUCLEO-H563ZI evaluation board. The firmware accepts typed commands over the ST-LINK virtual COM port and can reconfigure the active UART peripheral at runtime — changing baud rate and frame format while the system keeps running.

It was built incrementally across five labs, each adding one layer: non-blocking blink scheduling, interrupt-driven UART TX, button debounce with a 200 ms window, a TX message queue, and finally the full bidirectional console with command parsing and safe peripheral reconfiguration.

What the Console Does

After reset, the board starts a > prompt on USART3 at 115200 8N1. Three commands are supported: 

> help
Commands:
  help
  uart show
  uart set <baud> <8n1|8e1|8o1|8n2>

> uart show
UART console: 115200 8N1

> uart set 9600 8n1
Applying UART console: 9600 8N1. Update terminal now.
>

After uart set, the board drains any pending TX data, tears down USART3, rebuilds it with the new parameters, and restarts RX interrupt reception. The baud range is 1200–1000000; unsupported formats or out-of-range values print a usage error rather than silently failing.

Architecture

All I/O is non-blocking. There are no HAL_UART_Transmit blocking calls in the application path — every byte goes through one of two interrupt-driven data structures.

TX path ────────────────────────────────────────────────────── LogUart("msg") ├─ copy into TX queue (circular buffer, depth 16) └─ if UART idle → HAL_UART_Transmit_IT() HAL_UART_TxCpltCallback() └─ advance queue tail → start next message RX path ────────────────────────────────────────────────────── HAL_UART_RxCpltCallback() ├─ push byte into RX FIFO (circular buffer, depth 32) └─ restart HAL_UART_Receive_IT() immediately Main loop └─ ConsoleProcessRx() ├─ pop bytes from FIFO ├─ echo printable chars via LogUart() ├─ backspace removes last char └─ Enter → ConsoleProcessCommand()
The RX FIFO decouples the ISR from the main loop — the ISR stays short (push + restart), and the main loop handles all string building and parsing without dropping bytes during longer TX operations.

Interrupt-safe TX queue

The TX queue is shared between the main loop (writers) and the TX-complete ISR (reader). The critical section is kept as short as possible — just an index read, a pointer snapshot, and a flag set before re-enabling interrupts: 

static void UartLogStartTxIfIdle(void)
{
  __disable_irq();
  if ((g_uart_tx_busy != 0U) || (g_uart_log_head == g_uart_log_tail))
  {
    __enable_irq();
    return;                          /* nothing to send */
  }
  g_uart_tx_busy = 1U;
  tail_local = g_uart_log_tail;
  len = (uint16_t)strlen(g_uart_log_queue[tail_local]);
  __enable_irq();

  HAL_UART_Transmit_IT(&huart3,
      (uint8_t *)g_uart_log_queue[tail_local], len);
}

Safe UART reconfiguration

Changing UART settings while bytes are in-flight corrupts the link. The reconfiguration sequence calls UartWaitForTxDrain() before touching the peripheral, then tears it down and rebuilds it cleanly: 

static HAL_StatusTypeDef ConsoleApplyUartConfig(
    uint32_t baud_rate, uint32_t word_length,
    uint32_t parity,    uint32_t stop_bits)
{
  HAL_UART_AbortReceive_IT(&huart3);    /* stop RX interrupt */
  HAL_UART_DeInit(&huart3);            /* tear down */

  huart3.Init.BaudRate   = baud_rate;
  huart3.Init.WordLength = word_length;
  huart3.Init.Parity     = parity;
  huart3.Init.StopBits   = stop_bits;
  /* ... remaining Init fields ... */

  HAL_UART_Init(&huart3);

  /* STM32H5 FIFO settings must be restored after reinit */
  HAL_UARTEx_SetTxFifoThreshold(&huart3, UART_TXFIFO_THRESHOLD_1_8);
  HAL_UARTEx_SetRxFifoThreshold(&huart3, UART_RXFIFO_THRESHOLD_1_8);
  HAL_UARTEx_DisableFifoMode(&huart3);

  StartConsoleRx();                    /* restart RX interrupt */
  return HAL_OK;
}
The STM32H5 series has hardware FIFO on USART peripherals. After HAL_UART_Init the FIFO thresholds and mode must be restored explicitly — they are not preserved by the HAL reinit path. Missing this step causes the peripheral to come up in an unexpected state. It required a careful read of the reference manual to track down.

Command parser

The parser tokenizes a local copy of the line with strtok and validates each field before applying any change. Bad inputs get a specific error message rather than a silent failure: 

baud_rate = strtoul(baud_token, &end_ptr, 10);
if ((*baud_token == '\0') || (*end_ptr != '\0')
    || (baud_rate < 1200U) || (baud_rate > 1000000U))
{
  LogUart("Baud must be an integer in the range 1200..1000000.\r\n");
  return;
}

if (ConsoleTryParseFormat(format_token, &word_length,
                          &parity, &stop_bits, &format_name) == 0U)
{
  LogUart("Format must be one of: 8n1, 8e1, 8o1, 8n2.\r\n");
  return;
}

Build & Flash

Developed in STM32CubeIDE on the NUCLEO-H563ZI board (ST-LINK/V3 on-board).

  • Import the project: File → Import → Existing Projects into Workspace
  • Build: Project → Build All
  • Flash: Run → Debug (or Run)
  • Open a serial terminal at 115200 8N1 on the ST-LINK VCP port
  • Reset the board — the > prompt should appear

What I Learned

  • Structuring a circular queue so ISR and main loop share it safely with minimal critical section time
  • The STM32H5 FIFO quirk: HAL_UART_Init does not restore extended FIFO settings — they must be set again after every reinit
  • How to drain a TX queue before a peripheral teardown without spinning with interrupts disabled
  • Defensive input parsing: reject bad baud values, reject extra tokens, always print actionable error messages

Possible Next Steps

  • Store UART settings in flash so they survive a power cycle
  • Switch from one-byte-at-a-time RX interrupts to DMA + idle-line detection for higher throughput
  • Add a factory reset command to return to 115200 8N1
  • Add a dropped-message counter to detect TX queue saturation under load
